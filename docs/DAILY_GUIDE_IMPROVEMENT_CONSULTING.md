# 데일리 가이드 개선 컨설팅: 고정 문구 → 매일 기준 값 기반 문구

사용자가 실제로 도움이 되도록, **고정 문구** 대신 **매일 기준이 되는 값**에 맞춰 summary / do / avoid / relationships가 바뀌도록 하는 방향을 정리했습니다.

---

## 1. 현재 문제

- **avoid**, **relationships**는 코드에서 항상 동일한 문자열.
- **summary**, **do**는 Life Profile에 값이 있을 때만 개인화되고, 없으면 한 가지 기본 문구만 노출.
- **날짜·에너지 지수·최근 기록 추이** 등 “오늘”에 따라 문구가 바뀌지 않음.

→ 사용자 입장에서는 “매일 같은 조언”처럼 느껴질 수 있고, 실제 도움도 제한적입니다.

---

## 2. 매일 기준이 되는 값 (이미 있거나 계산 가능)

| 기준 값 | 의미 | 출처 |
|--------|------|------|
| **targetDate** | 조회하는 날짜 | API `date` 쿼리 또는 오늘 |
| **dayOfYear** | 1년 중 몇 번째 날 (1~365) | `targetDate`로 계산 |
| **phase** | 8단계 중 오늘 구간 (0~7) | `dayOfYear % 8` (이미 phase_tag로 사용 중) |
| **energy_index** | 오늘 에너지 지수 (0~100) | Life Profile balance + 최근 기록 3건 보정 |
| **recentLogs** | 최근 기록 3건 | energy/emotion/focus 평균, 추이(상승/하락/유지) |
| **lifeProfile** | 개인 패턴 | recommendations, cycleDescription, patterns, strengths |

이 값들로 “오늘은 에너지가 높은 날 / 보통 / 낮은 날”, “최근 추세가 올라가는 중 / 내려가는 중”, “오늘 phase가 상승 구간 / 정점 / 회복 구간” 등을 구분할 수 있습니다.

---

## 3. 개선 방향: 기준 값에 따른 문구 선택/조합

사주·AI 용어는 노출하지 않고, **숫자·날짜·구간**만으로 문구를 정해도 됩니다.

### 3.1 summary (오늘 한 줄 요약)

- **energy_index 구간별**로 다른 문장을 두는 방식이 자연스럽습니다.
  - 예: 0~40 “에너지를 아껴 쓰는 하루가 좋습니다. 무리한 일정은 미루고 휴식과 가벼운 루틴을 권장합니다.”
  - 41~60 “에너지가 보통 수준입니다. 중요한 일은 한두 가지로 묶고, 나머지는 유연하게 조정하세요.”
  - 61~80 “활동하기 좋은 날입니다. 오전에 중요한 일을 처리하고 오후에는 회복할 여유를 두세요.”
  - 81~100 “에너지가 높은 날입니다. 결정이나 창의적 작업, 협업을 앞당기기 좋습니다.”
- **추가**: Life Profile의 `cycleDescription`이 있으면, 위 구간 문장 **앞뒤**에 한 문장만 붙이거나, 구간 문장과 **둘 중 하나**를 선택하도록 할 수 있습니다. (개인 패턴 반영 + “오늘” 반영)

→ 요약은 “오늘 에너지 수준 + (선택) 개인 리듬 설명”으로 매일 달라지게 할 수 있습니다.

### 3.2 do (오늘 적합한 활동)

- **기본**: Life Profile의 `recommendations`가 있으면 그대로 사용 (이미 구현).
- **보강**: `energy_index` 구간별로 **추가/대체** 문장을 정의해 두고, 오늘 지수에 맞는 2~3개를 골라 `do` 목록 앞에 넣거나, recommendations와 섞어서 최대 5개 이내로 반환.
  - 예: 낮은 날 → “가벼운 산책”, “단기 작업만 하기”, “저녁 일찍 쉬기”  
  - 높은 날 → “중요 결정 하기”, “미뤄둔 일 처리”, “팀 미팅이나 협업”
- **phase**를 쓰고 싶다면: phase 0~1(새벽·상승), 2~3(정점·유지), 4~5(하강·저점), 6~7(회복·준비)처럼 묶어서 “오늘은 상승 구간이라 무리하지 않는 범위에서 계획 실행” 같은 한 문장을 do에 넣을 수 있습니다.

→ “개인 추천(리듬 타입)” + “오늘 에너지/phase”가 결합된, 매일 조금씩 다른 do 목록이 됩니다.

### 3.3 avoid (피해야 할 행동)

- 현재는 **완전 고정**이므로, **energy_index 구간별**로 다른 목록을 두는 것이 좋습니다.
  - 예: 0~40 → “과도한 업무 스케줄”, “새로운 큰 결단”, “늦은 밤까지 일하기”  
  - 41~60 → “한꺼번에 많은 일 맡기”, “중요한 약속을 오후 늦게 잡기”  
  - 61~80 → “과도한 업무 스케줄”, “충분한 휴식 없이 연속 일정”  
  - 81~100 → “에너지를 과하게 쓰는 밤늦은 약속”, “충분한 휴식 건너뛰기”
- **최근 기록 추이**가 “계속 낮음”이면 회복 쪽 조언(avoid에 “무리한 일정” 강조)을 더 강하게 줄 수 있습니다.

→ avoid도 “오늘 에너지 수준 + (선택) 최근 추이”에 따라 매일 바뀌게 할 수 있습니다.

### 3.4 relationships (인간관계 대응 팁)

- **고정 한 문장** 대신, **energy_index** 또는 **patterns 기준 “오늘 시간대”의 emotion/focus**를 이용해 2~3가지 문장 중 하나를 고르는 방식이 좋습니다.
  - 예: energy_index 높음 → “협업·대화가 잘 통하는 날입니다. 팀 프로젝트나 중요한 대화를 배치하기 좋습니다.”  
  - 보통 → “오늘은 협력적인 대화가 잘 통할 시기입니다. 무리한 요청보다는 조율과 소통에 집중하세요.”  
  - 낮음 → “에너지를 아껴야 하는 날입니다. 필수 대화만 짧게 하고, 큰 결정이나 갈등 대하는 대화는 미루는 편이 좋습니다.”
- Life Profile의 **strengths**가 있으면, “당신의 강점인 OO을 활용해 대화해 보세요”처럼 한 문장만 붙여도 개인화됩니다. (단, strength 문구에 사주 용어가 없도록 저장/선택 시 제어 필요)

→ relationships도 “오늘 에너지(또는 오늘 시간대 emotion)”에 따라 매일 달라지고, 선택적으로 개인 강점만 언급할 수 있습니다.

---

## 4. 구현 시 권장 사항

1. **energy_index 구간**  
   - 0–40 / 41–60 / 61–80 / 81–100 네 구간으로 나누고, 각 구간별로 summary / do 보강 / avoid / relationships용 **문장 풀**을 한 곳(예: `energyScoreFromProfile.js` 상단 상수 또는 별도 `dailyGuideCopy.js`)에 정의.  
   - 사주·명리 용어는 넣지 않고, “에너지”, “휴식”, “계획”, “협업” 등만 사용.

2. **computeDailyGuide 시그니처**  
   - `(lifeProfile, recentLogs)`에 **targetDate**(또는 `dayOfYear`, `phase`)를 넘기고,  
   - 내부에서 `energy_index` 계산 후 구간을 정하고,  
   - 해당 구간의 문장만 골라서 반환.  
   - avoid/relationships는 **항상** 이 구간 기반으로 반환(고정 문자열 제거).

3. **Life Profile과의 결합**  
   - summary: 구간 문장 + (있으면) `cycleDescription` 한 문장.  
   - do: `recommendations` 우선, 부족하거나 없으면 구간별 do 목록에서 채우기.  
   - relationships: 구간 문장 + (있으면) strengths 기반 한 문장만 추가.

4. **데이터 부족 시**  
   - Life Profile 없음 → energy_index만 75 등 기본값으로 두고, **구간은 “41~60” 또는 “61~80”**으로 고정해 해당 구간 문장 사용.  
   - 기록 없음 → 추이 없음으로 두고, 에너지 구간만으로 문구 결정.

5. **문구 관리**  
   - 나중에 문구만 수정할 수 있도록, **키(예: summary_low, do_high, avoid_mid, relationships_low)** 와 **한글 문장**을 매핑한 객체나 테이블로 두고, 코드에서는 키만 참조하는 방식이 유지보수에 유리합니다.

---

## 5. 정리

- **매일 기준 값**: `targetDate` → `dayOfYear`, `phase`, (이미 쓰는) `energy_index`, `recentLogs`, `lifeProfile`.
- **개선 방향**:  
  - **summary** → energy_index 구간별 문장 + (선택) cycleDescription.  
  - **do** → recommendations 기본 + energy_index(·phase) 구간별 보강.  
  - **avoid** → energy_index 구간별 목록(고정 1세트 제거).  
  - **relationships** → energy_index(또는 오늘 시간대 emotion) 구간별 문장 + (선택) strengths 한 줄.
- 이렇게 하면 **고정 문구보다 “오늘” 기준 값에 맞게** 바뀌고, 사용자에게 **실제로 도움이 될 만한 내용**으로 보이도록 개선할 수 있습니다.

원하시면 다음 단계로, 위 구간별 문장 예시를 코드에 넣는 단위(파일/함수)와 `computeDailyGuide` 수정 초안까지 이어서 정리해 드리겠습니다.
